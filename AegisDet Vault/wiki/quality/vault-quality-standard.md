# Vault Quality Standard

A high-quality note must be useful without opening ten unrelated placeholders. It should state its type, current status, specific purpose, project decision or hypothesis, implementation/evaluation method, and related evidence.

## Prohibited note patterns

- title repeated as the entire summary;
- generic checklists copied to every topic;
- claims without a source or experiment;
- completed status without evidence;
- unresolved wiki links;
- duplicate notes with ambiguous names;
- code pasted into wiki as the canonical implementation.

## Audit gates

The release audit checks file count, empty notes, unresolved links, duplicated stems, repeated boilerplate, Python syntax, unit tests, required control files, and ZIP root name.
